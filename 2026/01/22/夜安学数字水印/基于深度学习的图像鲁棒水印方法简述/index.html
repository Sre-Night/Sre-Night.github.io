基于深度学习的图像鲁棒水印方法简述

发表于2026-01-22|更新于2026-02-11|夜安学数字水印
|总字数:2.3k|阅读时长:7分钟|浏览量:

一、研究背景与意义

  1. 核心需求：数字媒体传播便捷性与版权保护需求的矛盾，数字水印技术通过在媒体中隐藏所有权 / 指纹信息，实现确权、溯源与分发跟踪，解决未经授权下载分发导致的权益损害问题。
  2. 技术演进：传统鲁棒水印依赖人工设计的固定嵌入空间（如 DCT、DWT、SVD 等变换），适应性差；深度学习技术可让模型自动学习稳定嵌入空间，并通过对抗学习提升抗攻击能力，成为主流研究方向。

二、核心技术框架：DDH 与 UDH

​ 两类框架是深度学习图像水印的基础架构，两者主要的区别在于DDH（Dependent Deep Hiding）编码器的输入和秘密信息及载体有关，而UDH（Universal Deep Hiding）仅与秘密信息有关，且均需通过反向传播优化隐藏网络（H）与提取网络（R），在训练中需插入失真层保证鲁棒性。

1.DDH框架

​ 如下图所示，左图框架主要由三部分组成，预处理网络（P）、编码器（H）和解码器（R）。首先对秘密信息进行预处理，得到秘密信息特征图，然后将特征图与载体图像C进行拼接后输入到H中，最终生成嵌入信息后的图像C’。解码时，利用R恢复出C’中嵌入的信息S’。

​ 右图中的框架和左图类似，差别在于S不需要进行预处理。若秘密信息为二进制字符串，通常将其重复多次以达到和C相同的大小，或将其重塑成C的形状后再提高到C的大小，然后将其与C拼接输入到H中完成信息嵌入。

2.UDH框架

​ 如下图所示，框架主要由H和R组成，H只编码秘密信息S，而与载体图像C无关。反向传播时只影响S如何编码而不对C进行修改。该框架在嵌入信息时，直接将H生成的秘密信息特征Se与C相加。当需要在多张载体图像Ci中嵌入相同的S时，使用该框架只需利用H生成一次Se即可，相比于DDH减少了很多工作量。解码时，可直接由R恢复出秘密信息S’。

3.框架总结

维度 Dependent Deep Hiding（DDH）框架 Universal Deep Hiding（UDH）框架
**编码器输入 ** 秘密信息 S + 载体图像 C 仅秘密信息 S
关键特性 载体适应性强，编码与 C 高频纹理深度融合 复用性高，同一 Se 可嵌入多载体，批量处理效率高
场景适配性 纹理丰富载体（如自然照片）：隐蔽性与恢复精度更优 光照变化 / 平滑扰动场景（如户外图像）：鲁棒性更优
编码特征 秘密信息编码为高频信息（将秘密信息利用UDH的H编码与DDR的R解码、DDH的H编码与UDH的R解码） 编码具跨信道性（多色彩通道共享编码）、局部性（集中局部区域）、秘密信息编码为高频信息
反向传播 梯度回传路径复杂（需从 R→C’→H，依赖 C 的特征梯度） 梯度路径简单（无 C 向 R 的梯度传递），训练更易稳定

两类框架在嵌入水印时还需在H和R之间插入失真层，以保证水印的鲁棒性。

三、研究方向

​ DNN 在鲁棒水印中的应用已从 “局部辅助”演进为 “端到端训练”，核心网络框架由四部分构成：

  • 编码器（encoder）：负责水印嵌入；
  • 解码器（decoder）：负责水印提取；
  • 判别器（discriminator）：基于对抗网络，保障含密载体的视觉质量；
  • 失真层（distortion）：模拟各类攻击，提升水印鲁棒性。

1.水印形式

根据水印形式不同划分两种情况，一种是二进制字符串，另一种是图像

  • 二进制字符串水印：

    • 信道编码策略：通过编码产生冗余，纠正传输中的错误；
    • 特征提取策略：利用一个特别的信息编码网络提取字符串的特征，将一个较短特征嵌入到载体中。

  • 图像水印：直接将图像作为秘密信息，通过 DDH/UDH 等框架完成嵌入与提取。

2. 攻击处理：不可微压缩（如 JPEG）

​ 攻击主要分为两种，已知攻击和未知攻击。针对已知攻击目前主要有两种解决方法，一种是生成真实攻击失真数据，缺点是代价太大。另一种是模拟攻击数据。

（1）JEPG压缩：

  • 色彩空间转换：将图像从 RGB（红 / 绿 / 蓝）转为 YCbCr（亮度 Y + 色度 Cb/Cr）—— 人眼对色度不敏感，后续会对 Cb/Cr 做更大幅度压缩；

  • 分块处理：将图像切割成 8×8 的像素块（核心操作），后续所有处理都基于这个最小单元；

  • DCT 变换：把每个 8×8 像素块从 “空间域”（像素值）转到 “频率域”—— 变换后得到 64 个 DCT 系数：

    • 左上角是低频系数（代表块的整体亮度 / 轮廓），对视觉至关重要；
    • 右下角是高频系数（代表块的细节 / 纹理 / 噪声），人眼不敏感；

  • 量化（核心有损步骤）：用预设的 “量化表” 对 64 个 DCT 系数做除法 + 舍入：

    • 量化表对高频系数的量化步长更大，导致高频系数被 “清零” 或大幅缩小；
    • 舍入是离散操作，无连续梯度，这也是深度水印模型反向传播时梯度中断的根

  • Z 字形扫描：按 “低频→高频” 的顺序排列量化后的系数，方便后续编码；

  • DC 系数预测：对每个块的低频 DC 系数做差分编码，即不需要存储每个块的 DC 系数绝对值，只需存储当前块 DC 系数 - 前一个块 DC 系数的差值，大幅减少数据量；

  • AC 系数游程编码：对高频 AC 系数（大量为 0）做 “连续 0 的个数 + 后续第一个非 0 系数的数值” 的编码，进一步压缩，剩余 AC 系数全 0，用EOB（End of Block） 标记；

  • 熵编码：用霍夫曼编码 / 算术编码对上述数据做无损压缩，生成最终的 JPEG 文件。

（2）解决方案：

JPEG 压缩的量化 / 舍入操作是核心挑战，主流解决方案具体包括：

解决方案 核心思路
可微压缩模拟 用连续函数近似量化 / 舍入，保留梯度以实现端到端训练
双阶段训练 阶段 1：无压缩训练 H/R 保证基础性能；
阶段 2：固定 H，仅训练 R 适应压缩失真，减轻梯度中断影响
失真层优化 训练时插入模拟攻击层（如 JPEG-Mask 舍弃高频、JPEG-Drop 渐进滤除系数），让模型学习攻击特性
攻击分配策略 每批数据分组含所有攻击类型，加速网络收敛
未知攻击应对 对抗学习；构建不变空间（通过全连接层，将编码器输出映射到 N 通道，解码器基于该空间解码）

3. 注意力机制融合

两类注意力机制分别优化鲁棒性与视觉质量，与对话中技术方向完全匹配：

  • 反向梯度注意力（鲁棒性提升）：生成注意力模版作为权重调制 C，根据解码损失动态更新模版，优先将水印嵌入抗攻击的高频纹理区域。
  • 视觉敏感区域注意力（不可感知性优化）：通过注意力网络定位人类视觉不敏感区域（如边缘、纹理），仅在这些区域嵌入水印，避免平滑区域失真。

四、总结与展望

1.总结

​ 衡量水印方法的三个关键因素是容量、不可感知性和鲁棒性。一个好的水印模型应能保证在视觉上不可感知，且可以对抗各种组合攻击（噪声、JPEG压缩、裁剪等）。基于DNN的鲁棒图像水印方法已取得了不错的进展，但仍然存在一些问题有待我们进一步研究.

2.现存关键问题：

  • 未知攻击与组合攻击抵御能力不足，现有方案对多类型、不同强度的组合攻击适配性有限，难以满足现实应用需求；
  • 不可微压缩（如 JPEG）的处理方案仍有优化空间，现有可微模拟、分离式训练等方法的解码准确率尚未达到理想水平；
  • 传统优化策略与 DNN 框架的融合不足，传统水印技术中成熟的人类视觉系统模型等，尚未有效融入深度学习架构以进一步提升不可感知性。
