进程的描述与控制

发表于2025-11-25|更新于2025-11-25|Sre-Night学操作系统
|总字数:2.5k|阅读时长:7分钟|浏览量:

一、前驱图和程序执行

1.程序的顺序执行

  • 一个较大的程序通常是由若干个程序段组成。
  • 程序在执行时，必须按照某种先后次序诸葛执行，仅当前一操作执行完后，才能执行后续操作。

2.前驱图

  • 有向无循环图，用于描述进程之间执行的先后顺序
  • 节点表示进程或程序段，有向边表示前驱关系

3.程序并发执行

  • 采用多道程序技术，将多个程序同时装入内存，使之并发运行

4.程序并发执行的特征

  • 间断性
    • 并发程序之间相互制约
    • 执行—-暂停执行—-执行
  • 失去封闭性
    • 多个程序共享全集资源
    • 执行状态受外界因素影响
  • 不可再现性
    • 程序经过多次执行后，虽然其执行时的环境和初始条件都相同，但得到的结果却各不相同

二、进程的描述

1.进程的定义和特征

  • 定义：进程是进程实体的运行过程，是系统进行资源分配和调度的独立单位
  • 进程控制块(PCB)：专门的数据结构，与进程一一对应
  • 特征：
    • 动态性（最基本的特征）：具有生命期
    • 并发性：一段时间内同时运行
    • 独立性：进程实体是一个能独立运行的基本单位，是系统进行资源分配和调度的基本单位
    • 异步性：按照各自独立的，不可预知的速度向前推进

2.进程的状态及转换

  • 就绪状态
  • 执行状态
  • 阻塞状态
  • 创建状态
  • 终止状态

引入挂起状态的状态转换

3.PCB

  • 定义PCB是进程的一部分，是操作系统中最重要的记录型数据结构，是进程存在的唯一标志，常驻内存

  • 作用

    • 作为独立运行基本单位的标志
    • 能实现间断性运行方式
    • 提供进程管理所需要的信息
    • 提供进程调度所需要的信息
    • 实现与其他进程的同步和通信

  • 表示信息

    • 进程标识符
    • 处理机状态
    • 进程调度信息
    • 进程控制信息

  • 组织方式

    • 线性方式

    • 链接方式

    • 索引方式

三、进程控制

1.进程控制

  • 进程管理最基本的功能
  • OS内核中原语实现
    • 进程创建
    • 进程终止
    • 进程阻塞与唤醒
    • 进程挂起与激活

2.进程创建

  • 进程具有层次结构
  • 引起事件：用户登录、作业调度、提供服务、应用请求
  • 进程图：（树）
  • 进程创建过程
    • 申请空白PCB
    • 分配所需资源
    • 初始化PCB
    • 插入就绪队列

3.进程终止

  • 引起事件：正常事件、异常结束、外界干预
  • 终止过程：
    • 根据被终止进程的标识符，从PCB集合中检索出该进程的PCB，从中读出该进程的状态
    • 若被终止进程处于执行状态，应立即终止该进程的执行，并设置调度标志为真，用于指示该进程被终止后应重新进行调度
    • 若该进程还有子孙进程，还将所有子进程予以终止
    • 将该进程的所拥有的全部资源，归还给父进程或系统
    • 将被终止进程PCB从所在队列中移去

4.进程的阻塞与唤醒

  • 引起进程阻塞与唤醒的事件：向系统请求共享资源失败；等待某种操作的完成；新数据尚未到达；等待新任务的到达
  • 进程阻塞过程
    • 阻塞原语Block()。
    • 一种主动行为
    • 具体过程：停止执行；状态由执行改为阻塞；将PCB插入阻塞队列
  • 进程唤醒过程
    • 唤醒原语Wakeup()
    • 具体过程：从阻塞队列中移出；状态由阻塞改为就绪；将PCB插入就绪队列
    • 必须成对使用Block()和Wakeup()

5.进程的挂起与激活

  • 进程挂起
    • Suspend()原语
    • 执行过程
  • 进程激活
    • Active()原语
    • 执行过程

四、进程通信

1.概念

  • 进程通信是指进程之间的信息交换
  • 低级进程通信：进程的同步和异步
    • 效率低
    • 通信对用户不信任
  • 高级进程通信
    • 使用方便
    • 高效地传送大量数据

2.类型

  • 共享存储器系统
    • 基于共享数据结构的通信方式（效率低）
    • 基于共享存储区的通信方式（高级）
  • 管道通信
    • 管道：用于连接一个读进程和一个写进程以实现它们之间通信的一个共享文件，又名**”pipe文件“**
    • 管道机制的协调能力：互斥、同步、对方是否存在
  • 消息传递系统
    • 直接通信方式
      1. 发送原语：send(receiver, message)
      2. 接收原语：receive(sender, message)
    • 间接通信方式**（通过邮箱）**
      • 信箱结构
      • 消息的发送和接收
        1. 发送原语：send(mailbox, message)
        2. 接收原语：receive(mailbox, message)
      • 信箱类型：私用信箱，公用信箱，共享信箱
  • 客户机—服务器系统
    • 嵌套字
    • 远程过程调用和远程方法调用
  • linux进程通信方式：管道、信号、消息队列、共享内存、信号量、嵌套字

五、线程的基本概念

1.概念

  • 引入进程的目的：
    • 使多个程序并发执行
    • 提高资源利用率和系统吞吐量
  • 两个基本属性
    • 进程是一个可拥有资源的独立单位
    • 进程是一个可独立调度和分派的基本单位
  • 引入线程的目的
    • 减少程序并发执行时所付出的时空开销
    • 使OS具有更好的并发性
    • 适用于SMP的计算机系统
  • 引入线程后
    • 进程是拥有资源的基本单位（传统进程称为重型进程）
    • 线程作为调度和分派的基本单位（又称轻型进程）
对比维 度 进程（Process） 线程（Thread）
1. 资源分配单位 操作系统资源分配的基本单位（独立内存、文件句柄等） 不独立分配资源，共享所属进程的全部资源，仅拥有一点保证独立运行的资源
2. 调度执行单位 非调度基本单位（操作系统不直接调度进程，而是调度其内部线程） 操作系统任务调度的基本单位（CPU 直接分配时间片给线程），同一进程中，线程的切换不会引起进程的切换
3. 内存空间 每个进程有独立的地址空间（进程 A 的内存不能直接访问进程 B 的内存） 线程共享进程的地址空间，线程间可直接访问共享内存
4. 上下文切换开销 开销大（切换时需保存 / 恢复进程的内存、文件句柄等所有资源状态） 开销小（仅需保存 / 恢复线程的栈、程序计数器等少量状态）
5. 通信方式 需依赖操作系统提供的 “进程间通信（IPC）” 机制（如管道、消息队列、共享内存） 可直接通过共享内存通信（如线程间共享进程内的变量），无需操作系统介入
6. 稳定性 进程间相互独立，一个进程崩溃不影响其他进程（如微信崩溃不会导致 Chrome 关闭） 线程属于同一进程，一个线程崩溃可能导致整个进程崩溃（如 Chrome 某标签页线程崩溃，可能导致浏览器闪退）
7. 创建 / 销毁开销 开销大（需申请内存、分配资源） 开销小（仅需创建栈和程序计数器，复用进程资源）
8. 并发性 进程之间可以并发执行 同一进程之间，多个线程可以并发

2.线程的状态和线程控制块

  • 线程状态：同进程
  • 线程控制块TCB
    • 线程标识符，一组寄存器，线程运行状态，优先级，线程专有存储区，信号屏蔽，堆栈指针

六、线程的实现

1.内核支持线程KST

在内核空间实现

  • 优点：
    • 在多处理机系统中，内核可同时调度同一进程的多个线程
    • 如一个线程阻塞，内核可调度其他线程（同一或其他进程）
    • 线程的切换比较快，开销小
    • 内核本身可采用多线程技术，提高执行速度和效率
  • 缺点：
    • 对用户线程的切换，开销较大

2.用户级线程ULT

在用户空间实现

  • 优点：
    • 线程切换不需要转换内核空间
    • 调度算法可以是进程专用的
    • 线程的实现与OS平台无关
  • 缺点
    • 系统调用的阻塞问题
    • 多线程应用不能利用多处理机进行多重处理的缺点

3.KST与ULT结合

  • 多对一模型
    • 多个用户级线程映射到一个内核线程
    • 多个线程不能并行运行在多个处理器上
    • 线程管理在用户态执行，因此是高效的，但一个线程阻塞，系统调用会导致整个进程的阻塞
    • 用于不支持内核线程的系统中
    • 例子：Solaris Green Threads GUN Portable Threads
  • 一对一模型
    • 每个用户级线程映射到一个内核线程
    • 比多对一有更好的并发性
    • 允许多个线程并行运行在多个处理机上
    • 创建一个ULT，需要创建一个KLT，效率较差
    • 例子：WIN 95/98/NT/XP/2000, Solaris 9and later, linux, OS2
  • 多对多模型
    • 多个用户级线程映射为相等或小于数目的内核线程
    • 允许操作系统创建足够多的KLT
